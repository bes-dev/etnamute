# PRD Schema

Formal specification for Etnamute Product Requirements Documents. Any PRD conforming to this schema can be used as input for headless builds.

---

## FORMAT

PRD is a markdown file with 12 numbered sections. All sections are required. Sections marked `(if monetized)` can be omitted or set to "N/A" for free apps.

---

## SCHEMA

```markdown
# App Specification: <app-name>

## 1. Overview

- **App Name**: <string, max 30 chars>
- **Slug**: <lowercase-kebab-case, used as directory name>
- **One-Line Pitch**: <single sentence>
- **Value Proposition**: <2-3 sentences>
- **App Category**: <App Store category>

## 2. Goals

### Primary Goals
- <goal 1>
- <goal 2>
- <goal 3>

### Non-Goals
- <what this app will NOT do>
- <features explicitly excluded>

## 3. Target User

- **Who**: <specific user description>
- **Pain Point**: <what frustrates them>
- **Current Solutions**: <what they use now and why it fails>
- **When**: <usage context — timing>
- **Where**: <mobile context — location>
- **Frequency**: <daily / weekly / occasional>

## 4. Core Features

### Feature 1: <name>
- **Purpose**: <why this feature exists>
- **User Action**: <what the user does>
- **System Response**: <what the app does>
- **Success State**: <how user knows it worked>

### Feature 2: <name>
<same structure>

### Feature 3: <name>
<same structure>

(3-5 features for MVP)

## 5. Monetization

**Model**: <free | freemium | subscription | one-time>

If free:
  No RevenueCat, no paywall, no premium features.

If monetized:
  - **Free Tier**: <what users get for free>
  - **Premium Tier**: <price + what it unlocks>
  - **Paywall Triggers**: <when paywall appears>

## 6. UX Philosophy

### Design Principles
- <principle 1>
- <principle 2>
- <principle 3>

### Key Screens
1. **Onboarding**: <description>
2. **Home**: <description>
3. **Core Action**: <description>
4. **Paywall**: <description — omit if free>
5. **Settings**: <description>

### Visual Style
- <style direction: minimal, colorful, dark, etc.>

## 7. Technical Constraints

- **Platform**: iOS + Android (Expo, latest stable SDK)
- **Language**: <en | language of request | multi: en, ru, ... >
- **Storage**: <expo-sqlite | AsyncStorage | both>
- **Sync**: <none | cloud backup | multi-device>
- **Backend**: <none | describe if needed>
- **Offline**: <what works offline>

## 8. Quality Bars

- **UI**: premium, domain-specific, consistent visual language
- **Performance**: <2s load, 60fps, low battery
- **Accessibility**: WCAG 2.1 AA, VoiceOver/TalkBack, 44pt touch targets

## 9. Market Research

### Market Overview
- <market size, trends, demand>

### Competitors
- **<Competitor 1>**: <strengths, weaknesses, pricing>
- **<Competitor 2>**: <strengths, weaknesses, pricing>

### Positioning
- **UVP**: <why choose this app>
- **Differentiation**: <what competitors miss>

## 10. ASO

- **Title**: <max 30 chars>
- **Subtitle**: <max 30 chars>
- **Description**: <full App Store description>
- **Keywords**: <comma-separated, max 100 chars total>

## 11. Deliverables

- [ ] Onboarding flow
- [ ] Home screen with core feature
- [ ] Paywall + RevenueCat (if monetized)
- [ ] Settings screen
- [ ] App icon + splash
- [ ] Privacy policy
- [ ] research/, aso/, marketing/ artifacts

## 12. Success Criteria

1. Core loop works end-to-end
2. Monetization works (if enabled)
3. All deliverables present
4. Ralph QA PASS (≥97%)
```

---

## VALIDATION RULES

A PRD is valid if:

1. All 12 sections present
2. `App Name` ≤ 30 characters
3. `Slug` is lowercase kebab-case
4. `Core Features` has 3-5 entries, each with Purpose/Action/Response/Success
5. `Monetization Model` is one of: `free`, `freemium`, `subscription`, `one-time`
6. If `Model` = `free` → no Premium Tier, no Paywall Triggers
7. If `Model` ≠ `free` → Premium Tier and Paywall Triggers are required
8. `Key Screens` has at least: Onboarding, Home, Core Action, Settings
9. `ASO Title` ≤ 30 chars, `ASO Subtitle` ≤ 30 chars, `ASO Keywords` ≤ 100 chars
10. `Market Research` section is substantive (not placeholder)
